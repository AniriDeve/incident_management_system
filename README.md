# Incident Management System

Небольшой backend-сервис для управления инцидентами.

Сделала этот проект, чтобы разобраться с базовыми вещами в backend-разработке: как устроен REST API, работа с базой данных и структура приложения на Spring Boot.

## Что реализовано

- создание инцидента
- получение списка инцидентов
- получение по id
- изменение статуса (NEW / IN_PROGRESS / RESOLVED)
- удаление

## Стек

- Java
- Spring Boot
- Spring Data JPA
- PostgreSQL
- REST API

## Как запустить

1. Установить PostgreSQL
2. Создать базу данных:
   incident_db

3. В application.yml указать свои данные для подключения к БД

4. Запустить проект (через IDE или командой)

## Пример запроса

POST /incidents
